# Use Cases

## 1) Full product revamp (0→100)
Use when you need a complete redesign, new information architecture, and modernized components/layouts.

Suggested sequence:
- `product-requirements-owner`
- `broader-design-architecture-overhaul`
- `nextjs-ui-architect`
- `nextjs-design-system-curator`
- `nextjs-ui-builder`
- `qa-test-architect`

## 2) Enterprise dashboard hardening
Use when dashboard complexity grows and design consistency/operability is weak.

Focus:
- dense data layout strategy
- reusable table/filter/action patterns
- permission and audit UX alignment

## 3) Marketing-to-product unification
Use when brand and product UI feel disconnected.

Focus:
- shared typography and spacing tokens
- consistent navigation paradigms
- common component language across route groups

## 4) Partial-scope modernization
Use when only a bounded module/page should be redesigned.

Focus:
- strict scope obedience
- dependency-aware local changes
- explicit out-of-scope reporting

## 5) PLAN.md-driven system architecture
Use when a project provides a `PLAN.md` and you need complete architecture planning before implementation.

Focus:
- parse PLAN.md into route and page inventory
- define feature decisions per page
- generate phased execution with risks and validation
