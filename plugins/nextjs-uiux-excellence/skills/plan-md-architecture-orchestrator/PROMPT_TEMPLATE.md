# PLAN.md System Planning Prompt Template

Use this prompt when a repository contains `PLAN.md` and the user requests full architecture/system planning.

## Prompt
"Read `PLAN.md` and create an implementation-ready system architecture. Build a complete page inventory, decide the required features for each page, and define route/layout architecture, auth boundaries, shared components, and validation strategy. Provide a phased execution roadmap and clearly mark unknowns that need clarification."

## Required output sections
1. Verified PLAN.md Facts
2. Architecture Blueprint
3. Page Inventory (new/keep/replace/deprecate)
4. Feature Matrix Per Page
5. Implementation Phases
6. Risks and Dependencies
7. Validation Plan
8. Clarification Questions
