# Workflow

1. Inspect the current UI foundation.
2. Identify duplicated or competing component systems.
3. Identify token drift across spacing, radius, shadow, color, and typography.
4. Decide whether to preserve, consolidate, or migrate.
5. Recommend the minimum strong component foundation.
6. Define how new UI work should inherit the design system.
7. Provide migration notes without inventing unnecessary refactors.
