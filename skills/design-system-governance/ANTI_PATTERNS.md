# Anti-Patterns

## Careless design-language mixing
Using several visual systems without a clear hierarchy.

## Primitive duplication
Creating many local variants of the same button, input, modal, badge, or card without a system reason.

## Token drift
Inconsistent spacing, radii, shadows, colors, or font scales.

## Visual-only improvement without system coherence
A surface looks better in isolation but makes the repo more fragmented overall.

## Accessibility regression
Choosing aesthetics that weaken keyboard support, contrast, labeling, or semantics.

## Over-engineered component abstraction
Creating elaborate design-system machinery before the product and repo justify it.
