# Anti-Patterns

## Fake product surfaces
Hardcoded records, dashboards, or activity feeds used to make a page look complete.

## Dead controls
Buttons, filters, tabs, or forms that appear functional but are not connected to real behavior.

## Generic AI-looking layouts
Flat, weakly composed UI with poor spacing, weak hierarchy, and no visual discipline.

## Visual polish masking fake functionality
A nice-looking page with no real data path or truthful empty/error states.

## Careless component mixing
Combining unrelated design systems or visual conventions without governance.

## Missing state integrity
No explicit loading, empty, error, unauthorized, or not-found states.

## Auth-unaware UI
Showing controls or management affordances without verified permission behavior.
