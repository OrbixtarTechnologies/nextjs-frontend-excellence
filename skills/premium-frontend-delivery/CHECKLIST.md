# Checklist

## Truthfulness
- verify data source
- verify action reality
- verify schema/contract assumptions
- prefer truthful empty states over fabricated content

## UI quality
- strong spacing rhythm
- deliberate typography hierarchy
- premium layout composition
- responsive quality
- accessible interactions

## State integrity
- loading
- empty
- error
- success
- unauthorized
- not-found

## System discipline
- reuse existing component patterns where strong
- improve weak patterns without careless drift
- keep visual language coherent
