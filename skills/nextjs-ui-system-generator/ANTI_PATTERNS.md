# Anti-Patterns

## Generic repo guidance
Creating frontend repo assets that could apply to any project.

## Duplicated local artifacts
Generating overlapping instructions, agents, prompts, and skills without clear boundaries.

## Premature path-specific sprawl
Creating many narrow frontend instruction files without a real repo need.

## Weak repo inspection
Generating local assets before understanding the repo’s router, design system, data patterns, and auth effects.

## Standardization theater
Claiming the repo now has a premium frontend system without creating enforceable local guidance.

## Architecture drift
Introducing a frontend operating model that conflicts with the repo’s real structure and conventions.
