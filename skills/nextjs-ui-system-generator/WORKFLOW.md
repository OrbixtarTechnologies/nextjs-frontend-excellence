# Workflow

## 1. Inspect and verify the repo
- determine whether the repo uses App Router or Pages Router
- identify frontend path ownership
- identify component-system and styling-system strength
- identify auth/permission effects on UI

## 2. Map the frontend operating surface
- determine where frontend instructions should live
- determine whether path-specific frontend instructions are justified
- determine whether frontend prompt files are justified
- determine whether dedicated frontend skills are justified

## 3. Decide the minimum strong local artifact set
Create only the assets that materially improve future Copilot behavior in this repository.

## 4. Generate the local frontend assets
Generated assets must be:
- repo-aware
- non-duplicative
- premium in quality expectations
- truthful in implementation expectations

## 5. Self-audit the generated system
Check for:
- duplicate scope
- contradictory rules
- generic wording
- weak repo adaptation
- shallow UI quality guidance
- missing validation expectations
