# Next.js UI System Generator

## Use this skill for
- Bootstrapping repo-level frontend Copilot customizations in a Next.js repo
- Standardizing weak or inconsistent frontend repos
- Creating repo-specific frontend instructions, skills, and prompts
- Establishing a premium design-system direction

## Core mandate
You must inspect the active repository and generate only the repo-level frontend Copilot assets that are justified by the repository.

## Non-negotiables
- no invented repo facts
- no fake UI
- no fake data
- no hallucinated API/schema usage
- no generic AI-looking interface patterns
- no shallow repo guidance
- no duplicated local artifacts
- no path-specific files unless justified

## Required inspection
Inspect and summarize:
1. app/router structure
2. page/layout conventions
3. component structure
4. styling stack
5. design-system usage
6. form patterns
7. validation patterns
8. data-fetching/state patterns
9. auth/permission effects
10. accessibility quality
11. responsive quality
12. repeated anti-patterns

## Repo-level assets this skill may generate
- `.github/instructions/frontend.instructions.md`
- `.github/instructions/ui-system.instructions.md`
- `.github/agents/frontend-ui.agent.md`
- `.github/prompts/design-ui-change.prompt.md`
- `.github/prompts/implement-premium-ui.prompt.md`
- `.github/skills/frontend-premium-delivery/SKILL.md`
- supporting frontend governance docs

Generate only what is justified.

## UI-kit strategy
Use the project’s current design system first.
If standardization is needed, prefer:
1. shadcn/ui
2. Radix UI
3. Headless UI where appropriate

## Quality bar
Any repo-level frontend system generated by this skill must enforce:
- truthful state handling
- premium visual quality
- consistent spacing and typography
- responsive discipline
- accessibility
- reusable components
- no fake completion language
